# Recursion 演算法詳解

## 一、什麼是 Recursion？

**Recursion（遞迴）** 是一種函數呼叫自身的程式設計技巧，將大問題分解成結構相同但規模較小的子問題，直到達到可直接求解的**基底情況（Base Case）**。  
核心想法是：**分而治之**，透過「定義問題與子問題的關係」與「終止條件」，用簡潔的程式碼表達複雜的邏輯，常見於樹、圖、回溯、分治等演算法。

---

## 二、演算法原理

### 基本概念

1. **Base Case（基底情況）**：遞迴的終止條件，不再呼叫自身，直接回傳結果。
2. **Recursive Case（遞迴情況）**：將問題分解為子問題，呼叫自身處理子問題，再合併結果。
3. **遞迴樹**：可將遞迴展開為樹狀結構，幫助分析時間與空間複雜度。
4. **避免無限遞迴**：必須確保每次遞迴都朝 Base Case 前進，否則會堆疊溢位。

### 常見變體

| 變體 | 說明 | 典型應用 |
|------|------|----------|
| **線性遞迴** | 每次只產生一個遞迴呼叫 | 階乘、費氏數列、鏈結串列遍歷 |
| **分支遞迴** | 每次產生多個遞迴呼叫 | 二元樹遍歷、Merge Sort、Quick Sort |
| **回溯** | 嘗試選擇 → 遞迴 → 撤銷選擇 | 全排列、子集、N 皇后 |
| **分治** | 分解 → 遞迴求解 → 合併 | Merge Sort、二元樹問題 |
| **尾遞迴** | 遞迴呼叫為函數最後一個操作 | 可被編譯器優化為迭代，減少堆疊 |

### 執行步驟（以階乘為例）

1. 定義 Base Case：`factorial(0) = 1` 或 `factorial(1) = 1`。
2. 定義遞迴關係：`factorial(n) = n * factorial(n - 1)`。
3. 呼叫 `factorial(n)` 時，若 `n <= 1` 回傳 1；否則回傳 `n * factorial(n - 1)`。
4. 遞迴會一路呼叫到 `factorial(0)` 或 `factorial(1)`，再依序回傳結果。

### 範例：階乘

`factorial(4)`：

- `4 * factorial(3)` → `3 * factorial(2)` → `2 * factorial(1)` → `1`（Base Case）
- 回傳：`1` → `2` → `6` → `24`

### 範例：鏈結串列兩數相加（遞迴）

LeetCode 2：`l1 = [2,4,3]`，`l2 = [5,6,4]`：

- 處理個位：`2+5=7`，carry=0，遞迴處理 `[4,3]` 與 `[6,4]`
- 處理十位：`4+6=10`，carry=1，遞迴處理 `[3]` 與 `[4]`
- 依此類推，最後合併成 `[7,0,8]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 樹與圖的遍歷 | 二元樹前中後序、DFS |
| 分治法 | Merge Sort、Quick Sort、二元樹問題 |
| 回溯 | 全排列、子集、組合、N 皇后 |

| 數學定義遞迴 | 費氏數列、階乘、漢諾塔 |
| 子問題結構相同 | 問題可分解為相同形式的子問題 |

## 四、時間複雜度

- **線性遞迴**：O(n)，如階乘、單向鏈結串列遍歷。
- **分支遞迴（二元樹）**：O(n)，每個節點訪問一次。
- **分支遞迴（每次分兩半）**：O(n log n)，如 Merge Sort。
- **回溯（全排列）**：O(n!)，如 Permutations。
- **回溯（子集）**：O(2^n)，如 Subsets。
- 依遞迴樹的結構與分支數而定。

---

## 五、空間複雜度

- **空間複雜度**：O(遞迴深度)
- 遞迴呼叫會使用**呼叫堆疊**，深度即為空間消耗。
- 線性遞迴：O(n)，如 `factorial(n)` 深度為 n。
- 二元樹遞迴：O(h)，h 為樹高，最壞 O(n)。
- 尾遞迴可被優化為 O(1)（依語言與編譯器而定）。

---

## 六、與其他技巧的比較

| 技巧 | 時間複雜度 | 空間複雜度 | 適用條件 |
|------|------------|------------|----------|
| Recursion | 依問題而定 | O(深度) | 樹、圖、回溯、分治 |
| 迭代 | 同遞迴 | O(1) | 可改寫為迴圈時 |
| Two Pointers | O(n) | O(1) | 陣列、配對 |
| DP | O(n)～O(n²) | O(n) | 有重複子問題時，可取代遞迴 |
| 回溯 | O(2^n)～O(n!) | O(n) | 枚舉、組合、排列 |

---

## 七、總結

Recursion 是將大問題分解為子問題的經典技巧，適用於樹、圖、回溯、分治等場景。關鍵在於正確定義 **Base Case** 與 **遞迴關係**，並確保遞迴終止。需注意堆疊深度與空間消耗，必要時可改為迭代或搭配記憶化（Memoization）優化為動態規劃。
