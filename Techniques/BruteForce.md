# 暴力搜尋 演算法詳解

## 一、什麼是暴力搜尋？

**暴力搜尋（Brute Force）** 是一種直接枚舉所有可能解並逐一檢查的演算法策略。  
核心想法是：不依賴特殊資料結構或巧妙優化，而是**窮舉**所有候選答案，直到找到符合條件的解。實作簡單、邏輯直觀，但通常效能較差。

---

## 二、演算法原理

### 基本概念

1. **窮舉**：遍歷所有可能的候選解，不跳過任何選項。
2. **逐一檢查**：對每個候選解驗證是否滿足題目條件。
3. **無優化**：不利用資料特性（如有序性、重複性）來剪枝或加速。
4. **保證正確**：若解存在，暴力法一定能找到（可能較慢）。

### 常見變體

| 變體 | 說明 | 典型應用 |
|------|------|----------|
| **線性掃描** | 從頭到尾遍歷一次 | 找最大值、找目標值 |
| **雙層迴圈** | 枚舉所有 (i, j) 配對 | 兩數之和、子陣列和 |
| **多層迴圈** | 枚舉 k 元組 | 三數之和、組合枚舉 |
| **遞迴枚舉** | 遞迴產生所有組合/排列 | 全排列、子集、回溯 |

### 執行步驟（以兩數之和為例）

1. 外層迴圈 `i` 從 0 到 n-1。
2. 內層迴圈 `j` 從 i+1 到 n-1。
3. 若 `arr[i] + arr[j] == target`，回傳 `[i, j]`。
4. 若遍歷完無解，回傳空或 -1。

### 範例：兩數之和

陣列 `[2, 7, 11, 15]`，目標 9：

- `i=0, j=1`：`2+7=9` → 找到 `[0, 1]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 小規模資料 | n 很小時，暴力法可接受 |
| 無更好方法 | 問題本質需枚舉，難以優化 |
| 驗證正確性 | 作為其他解法的對照基準 |
| 實作簡單 | 快速寫出可運行的解法 |
| 面試暖身 | 先寫暴力解再優化 |

### 常見題型

1. **LeetCode 1**: Two Sum（暴力 O(n²)，可優化為 Hash O(n)）
2. **LeetCode 15**: 3Sum（暴力 O(n³)，可優化為排序+雙指標 O(n²)）
3. **LeetCode 53**: Maximum Subarray（暴力 O(n²)，可優化為 Kadane O(n)）
4. **LeetCode 121**: Best Time to Buy and Sell Stock（暴力 O(n²)）
5. **LeetCode 217**: Contains Duplicate（暴力 O(n²)）
6. **LeetCode 242**: Valid Anagram（暴力排序比較 O(n log n)）
7. **LeetCode 46**: Permutations（遞迴枚舉全排列）
8. **LeetCode 78**: Subsets（遞迴枚舉子集）

---

## 四、時間複雜度

- **線性掃描**：O(n)
- **雙層迴圈**：O(n²)
- **三層迴圈**：O(n³)
- **全排列**：O(n!)
- **子集**：O(2^n)
- 依枚舉規模而定，通常為多項式或指數級。

---

## 五、空間複雜度

- **空間複雜度**：通常 O(1)（不計輸入與輸出）
- 僅使用迴圈變數與少數暫存。
- 若使用遞迴，堆疊深度為 O(n)，空間 O(n)。
- 若需儲存所有解（如全排列），則為 O(解數量)。

---

## 六、與其他技巧的比較

| 技巧 | 時間複雜度 | 空間複雜度 | 適用條件 |
|------|------------|------------|----------|
| 暴力搜尋 | O(n²)～O(n!) | O(1) | 小規模、無更好方法 |
| Two Pointers | O(n) | O(1) | 有序陣列、配對 |
| Hash Table | O(n) | O(n) | 需快速查詢、無需排序 |
| 滑動窗口 | O(n) | O(1) | 連續子陣列/子字串 |
| 二分搜尋 | O(log n) | O(1) | 單一目標、有序 |

---

## 七、總結

暴力搜尋是最直觀的解法，實作簡單、邏輯清晰，但效能通常較差。適合小規模資料、驗證正確性，或作為優化解法的起點。遇到可優化的題型時，應考慮 Hash、雙指標、二分搜尋等技巧以降低時間複雜度。
