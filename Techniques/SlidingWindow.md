# Sliding Window 演算法詳解

## 一、什麼是 Sliding Window？

**Sliding Window（滑動窗口）** 是一種處理**連續子陣列**或**連續子字串**的演算法技巧，透過維護一個在資料上「滑動」的區間窗口，在 O(n) 內求解最長、最短或滿足條件的區間。  
核心想法是：利用**窗口內狀態的增量更新**，避免每次重新計算整個區間，從而將 O(n²) 降為 O(n)。

---

## 二、演算法原理

### 基本概念

1. **窗口**：由 `left` 與 `right` 指標界定的連續區間 `[left, right]`。
2. **滑動**：`right` 擴展窗口，`left` 收縮窗口，兩者皆單向移動。
3. **狀態維護**：用變數或 Hash 記錄窗口內的元素、計數、和等，隨窗口移動增量更新。
4. **單向移動**：`left` 與 `right` 最多各走 n 步，總時間 O(n)。

### 常見變體

| 變體 | 說明 | 典型應用 |
|------|------|----------|
| **固定長度** | 窗口大小固定為 k | 長度 k 的最大子陣列和 |
| **可變長度** | 窗口大小依條件變化 | 最長不重複子字串、最小覆蓋子字串 |
| **最大窗口** | 盡量擴展，不滿足時收縮 | 最長滿足條件的子陣列 |
| **最小窗口** | 滿足條件後盡量收縮 | 最小覆蓋子字串 |

### 執行步驟（以最長不重複子字串為例）

1. 初始化 `left = 0`，`right = 0`，Hash 記錄字元出現次數，`maxLen = 0`。
2. `right` 右移，將 `s[right]` 加入窗口，更新 Hash。
3. 若某字元出現次數 > 1，則 `left` 右移，移除 `s[left]`，直到窗口內無重複。
4. 更新 `maxLen = max(maxLen, right - left + 1)`。
5. 重複 2～4 直到 `right` 到達結尾。

### 範例：最長不重複子字串

字串 `"abcabcbb"`：

- `[a]` → `[ab]` → `[abc]` → 加入 `a` 重複，`left` 右移 → `[bca]` → ...
- 最長為 `"abc"` 或 `"bca"`，長度 3。

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 連續子陣列/子字串 | 要求區間必須連續 |
| 最長/最短滿足條件 | 求符合條件的最長或最短區間 |
| 固定長度區間 | 如長度 k 的最大和 |
| 無重複、最多 k 個等 | 窗口內元素需滿足約束 |
| 字串、陣列皆可 | 適用於一維序列 |

### 常見題型

1. **LeetCode 3**: Longest Substring Without Repeating Characters
2. **LeetCode 76**: Minimum Window Substring
3. **LeetCode 209**: Minimum Size Subarray Sum
4. **LeetCode 424**: Longest Repeating Character Replacement
5. **LeetCode 438**: Find All Anagrams in a String
6. **LeetCode 567**: Permutation in String
7. **LeetCode 904**: Fruit Into Baskets
8. **LeetCode 992**: Subarrays with K Different Integers
9. **LeetCode 1248**: Count Nice Subarrays（可轉為滑動窗口）

## 四、時間複雜度

- **時間複雜度**：O(n)
- `left` 與 `right` 各最多移動 n 次，每個元素最多被加入與移除窗口各一次。
- 若窗口內需 Hash 查詢，仍為 O(n)，因總操作次數與 n 同階。

---

## 五、空間複雜度

- **空間複雜度**：O(1) 或 O(k)
- 若用固定大小字元集（如 26 字母），Hash 為 O(1)。
- 若需記錄窗口內所有不同元素，最壞 O(n)，通常題目會限制字元集大小。

---

## 六、與其他技巧的比較

| 技巧 | 時間複雜度 | 空間複雜度 | 適用條件 |
|------|------------|------------|----------|
| 滑動窗口 | O(n) | O(1)～O(k) | 連續子陣列/子字串 |
| 暴力搜尋 | O(n²) | O(1) | 枚舉所有子陣列 |
| Two Pointers | O(n) | O(1) | 配對、回文（滑動窗口為其變體） |
| Hash Table | O(n) | O(n) | 計數、存在性（常與窗口搭配） |
| 二分搜尋 | O(log n) | O(1) | 單一目標、有序 |

---

## 七、總結

Sliding Window 是處理連續子陣列/子字串的經典技巧，透過雙指標與狀態的增量更新，在 O(n) 內求解最長、最短或滿足條件的區間。固定長度與可變長度是兩大類型，依題目選擇擴展或收縮策略。與 Hash 搭配可處理「無重複」「最多 k 種」等約束。
