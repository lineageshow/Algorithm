# Two Pointers 演算法詳解

## 一、什麼是 Two Pointers？

**Two Pointers（雙指標）** 是一種常用的演算法技巧，透過維護兩個指標（索引）在資料結構中移動，以達到搜尋、比對或遍歷的目的。  
核心想法是：利用資料的**有序性**或**單調性**，讓兩個指標從不同方向或不同起點移動，在 O(n) 或 O(n²) 內完成原本可能需要暴力搜尋的問題。

---

## 二、演算法原理

### 基本概念

1. **雙指標**：通常為 `left` 與 `right`，或 `slow` 與 `fast`，依題型不同而異。
2. **移動策略**：根據當前條件決定移動哪個指標、往哪個方向移動。
3. **終止條件**：兩指標相遇、越界或達成目標時結束。
4. **有序性**：多數題型假設陣列已排序，才能利用雙指標縮小搜尋範圍。

### 常見變體

| 變體 | 說明 | 典型應用 |
|------|------|----------|
| **對撞型** | 兩指標從頭尾往中間移動 | 兩數之和、三數之和、回文檢查 |
| **快慢型** | 兩指標同向移動，速度不同 | 鏈結串列環偵測、找中點 |
| **滑動窗口** | 兩指標維持固定或可變區間 | 子陣列和、最長不重複子字串 |
| **分離型** | 一指標遍歷，一指標指向「寫入位置」 | 原地移除元素、合併有序陣列 |

### 執行步驟（以對撞型為例）

1. 初始化 `left = 0`，`right = n - 1`。
2. 若 `left < right`，計算當前狀態（如 `arr[left] + arr[right]`）。
3. 根據結果決定：
   - 符合目標 → 記錄答案，視情況移動指標。
   - 結果過大 → 移動 `right` 向左（若升序）。
   - 結果過小 → 移動 `left` 向右。
4. 重複直到 `left >= right`。

### 範例：兩數之和（陣列已排序）

陣列 `[1, 2, 3, 4, 6]`，目標和為 6：

- `left=0, right=4`：`1+6=7 > 6` → `right--`
- `left=0, right=3`：`1+4=5 < 6` → `left++`
- `left=1, right=3`：`2+4=6` → 找到 `[2, 4]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 陣列已排序 | 可利用大小關係縮小搜尋範圍 |
| 找滿足條件的配對 | 兩數之和、三數之和等 |
| 回文檢查 | 從頭尾往中間比對 |
| 鏈結串列問題 | 環偵測、找中點、合併兩串列 |
| 子陣列/子字串 | 滑動窗口求最長、最短、滿足條件 |
| 原地修改 | 移除元素、去重、合併 |

## 四、時間複雜度

- **對撞型 / 分離型**：O(n)，每個元素最多被兩指標各訪問一次。
- **快慢指標**：O(n)，慢指標最多走 n 步，快指標最多 2n 步。
- **滑動窗口**：O(n)，每個元素最多被加入與移除窗口各一次。
- **三數之和等擴展**：O(n²)，外層固定一數，內層雙指標 O(n)。

---

## 五、空間複雜度

- **空間複雜度**：O(1)（不計輸入陣列）
- 僅使用常數個變數（兩個指標、幾個暫存變數）。
- 若需額外儲存結果（如答案陣列），則為 O(k)，k 為結果數量。

---

## 六、與其他技巧的比較

| 技巧 | 時間複雜度 | 空間複雜度 | 適用條件 |
|------|------------|------------|----------|
| Two Pointers | O(n) | O(1) | 有序陣列、配對、回文 |
| 暴力搜尋 | O(n²) | O(1) | 小規模、無更好方法 |
| Hash Table | O(n) | O(n) | 需快速查詢、無需排序 |
| 滑動窗口 | O(n) | O(1) | 連續子陣列/子字串 |
| 二分搜尋 | O(log n) | O(1) | 單一目標、有序 |

---

## 七、總結

Two Pointers 是處理陣列、字串、鏈結串列時常用的技巧，能將許多 O(n²) 問題降為 O(n)。關鍵在於善用資料的**有序性**與**單調性**，透過雙指標的移動策略避免不必要的枚舉。對撞型、快慢型、滑動窗口是三大常見變體，依題型選擇合適的實作方式。
