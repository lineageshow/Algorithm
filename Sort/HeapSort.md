# Heap Sort 演算法詳解

## 一、什麼是 Heap Sort？

**Heap Sort（堆積排序）** 是一種基於**二元堆積（Binary Heap）**資料結構的比較排序演算法。  
核心想法是：將陣列視為二元堆積，先建立 Max Heap（最大堆積），再反覆取出堆頂（最大值）與堆尾交換，並重新調整堆積，直到堆積縮小為空，即完成升序排序。

---

## 二、演算法原理

### 基本概念

1. **二元堆積**：完全二元樹，父節點 ≥ 子節點（Max Heap）或父節點 ≤ 子節點（Min Heap）。
2. **陣列表示**：索引 `i` 的父節點為 `(i-1)/2`，左子為 `2i+1`，右子為 `2i+2`。
3. **Heapify（堆積化）**：從某節點往下調整，確保以該節點為根的子樹符合堆積性質。
4. **Build Heap**：從最後一個非葉節點往上，依序對每個節點執行 Heapify，建立完整 Max Heap。

### 執行步驟

1. **建立 Max Heap**：從 `n/2 - 1` 往下到 0，對每個索引執行 Heapify，使整個陣列成為 Max Heap。
2. **排序**：從 `n-1` 往下到 1：
   - 將堆頂 `arr[0]`（當前最大值）與 `arr[i]` 交換。
   - 堆積有效長度減 1。
   - 對 `arr[0]` 執行 Heapify，恢復堆積性質。
3. 重複步驟 2 直到堆積長度為 1，陣列即為升序。

### 範例

陣列 `[4, 10, 3, 5, 1]`：

- 建立 Max Heap：`[10, 5, 3, 4, 1]`
- 交換：`arr[0]` 與 `arr[4]` → `[1, 5, 3, 4 | 10]`，Heapify → `[5, 4, 3, 1 | 10]`
- 交換：`arr[0]` 與 `arr[3]` → `[1, 4, 3 | 5, 10]`，Heapify → `[4, 1, 3 | 5, 10]`
- 依此類推
- 結果：`[1, 3, 4, 5, 10]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 需要 O(n log n) 保證 | 無最壞 O(n²)，效能穩定 |
| 原地排序 | O(1) 額外空間，不需額外陣列 |
| 不穩定排序可接受 | Heap Sort 為不穩定排序 |
| 優先佇列應用 | 堆積結構可同時用於排序與優先佇列 |
| 嵌入式或記憶體受限 | 不需遞迴或大量額外空間 |

### 常見題型

1. **LeetCode 215**: Kth Largest Element in an Array（可結合 Heap 或 Quick Select）
2. **LeetCode 347**: Top K Frequent Elements（Min Heap 維護 K 大）
3. **LeetCode 23**: Merge k Sorted Lists（Min Heap 合併）
4. **LeetCode 295**: Find Median from Data Stream（雙堆積）
5. **LeetCode 912**: Sort an Array

## 四、時間複雜度

- **時間複雜度**：O(n log n)，無論最佳、平均、最壞情況皆相同。
- Build Heap：O(n)（非 O(n log n)，因多數節點高度較低）。
- 每次取出堆頂並 Heapify：O(log n)，共 n-1 次，故排序階段為 O(n log n)。

---

## 五、空間複雜度

- **空間複雜度**：O(1)
- 原地操作，直接將陣列視為堆積，不需額外陣列。
- Heapify 可實作為迭代，不需遞迴堆疊。

---

## 六、與其他排序的比較

| 演算法 | 時間複雜度 | 空間複雜度 | 適用條件 |
|--------|------------|------------|----------|
| Heap Sort | O(n log n) | O(1) | 一般陣列、原地 |
| Quick Sort | O(n log n) 平均 / O(n²) 最壞 | O(log n) | 一般陣列 |
| Merge Sort | O(n log n) | O(n) | 一般陣列、需穩定排序 |
| Bubble Sort | O(n²) | O(1) | 小規模、教學用 |
| Cyclic Sort | O(n) | O(1) | 元素為 1～n 且不重複 |

---

## 七、總結

Heap Sort 在 O(n log n) 時間、O(1) 空間內完成排序，無最壞情況，且為原地排序。缺點是不穩定，且常數因子較大，實務上通常比 Quick Sort 慢。適合需要效能保證、記憶體受限或需結合堆積資料結構的場景。
