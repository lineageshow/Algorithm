# Merge Sort 演算法詳解

## 一、什麼是 Merge Sort？

**Merge Sort（合併排序）** 是一種基於分治法（Divide and Conquer）的比較排序演算法。  
核心想法是：將陣列不斷對半分割，直到每個子陣列只剩一個元素，再兩兩合併成有序陣列，最終得到完整排序結果。Merge Sort 是**穩定排序**。

---

## 二、演算法原理

### 基本概念

1. **Divide（分割）**：將陣列從中間分成左右兩半，遞迴分割直到子陣列長度為 1。
2. **Conquer（合併）**：將兩個已排序的子陣列合併成一個有序陣列，使用雙指標比較並依序放入結果。
3. **穩定性**：合併時若兩元素相等，優先取左側元素，可保持相對順序不變。

### 執行步驟

1. 若陣列長度 ≤ 1，直接回傳（已排序）。
2. 計算中間索引 `mid = (left + right) / 2`。
3. 對左半部 `[left, mid]` 遞迴執行 Merge Sort。
4. 對右半部 `[mid + 1, right]` 遞迴執行 Merge Sort。
5. 合併左右兩個已排序子陣列：用雙指標比較，較小者放入結果陣列，直到兩邊都取完。
6. 將合併結果寫回原陣列對應區間。

### 範例

陣列 `[38, 27, 43, 3, 9, 82, 10]`：

- 分割：`[38, 27, 43, 3]` | `[9, 82, 10]`
- 再分割：`[38, 27]` | `[43, 3]` 與 `[9, 82]` | `[10]`
- 再分割至單元素：`[38]` `[27]` `[43]` `[3]` `[9]` `[82]` `[10]`
- 合併：`[27, 38]`、`[3, 43]` → `[3, 27, 38, 43]`；`[9, 82]`、`[10]` → `[9, 10, 82]`
- 最終合併：`[3, 9, 10, 27, 38, 43, 82]`

---

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 需要穩定排序 | 相等元素保持原有相對順序 |
| 鏈結串列排序 | 不需隨機存取，Merge Sort 適合鏈結串列 |
| 外部排序 | 大檔案無法一次載入記憶體時，可分批排序再合併 |
| 時間複雜度可預測 | 無最壞情況，恆為 O(n log n) |
| 並行化 | 左右子陣列可平行遞迴，再合併 |

### 常見題型

1. **LeetCode 912**: Sort an Array
2. **LeetCode 148**: Sort List（鏈結串列排序）
3. **LeetCode 23**: Merge k Sorted Lists
4. **LeetCode 315**: Count of Smaller Numbers After Self（逆序對，可結合 Merge Sort）
5. **LeetCode 493**: Reverse Pairs

---

## 四、時間複雜度

- **時間複雜度**：O(n log n)，無論最佳、平均、最壞情況皆相同。
- 分割層數為 log n，每層合併總工作量為 O(n)，故整體 O(n log n)。

---

## 五、空間複雜度

- **空間複雜度**：O(n)
- 合併時需要額外陣列存放暫時結果。
- 遞迴呼叫堆疊為 O(log n)，但主要空間消耗來自合併用的輔助陣列。

---

## 六、與其他排序的比較

| 演算法 | 時間複雜度 | 空間複雜度 | 適用條件 |
|--------|------------|------------|----------|
| Merge Sort | O(n log n) | O(n) | 一般陣列、需穩定排序 |
| Quick Sort | O(n log n) 平均 / O(n²) 最壞 | O(log n) | 一般陣列 |
| Cyclic Sort | O(n) | O(1) | 元素為 1～n 且不重複 |
| Counting Sort | O(n + k) | O(k) | 元素範圍有限 |

---

## 七、總結

Merge Sort 是穩定排序，時間複雜度恆為 O(n log n)，無最壞情況。代價是需 O(n) 額外空間。適合需要穩定排序、鏈結串列排序或外部排序的情境。若只需一般陣列排序且可接受不穩定，Quick Sort 在實務上常因快取 locality 而較快。
