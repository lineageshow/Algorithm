# Cyclic Sort 演算法詳解

## 一、什麼是 Cyclic Sort？

**Cyclic Sort（循環排序）** 是一種針對「元素在固定範圍內且連續」陣列的原地排序演算法。  
核心想法是：若陣列長度為 n，且元素為 1～n（或 0～n-1）且不重複，則每個元素都有唯一「正確位置」，只要把每個元素放到對應索引，就能完成排序。

---

## 二、演算法原理

### 基本概念

1. **前提條件**：陣列長度為 n，元素為 1～n（或 0～n-1）且不重複。
2. **正確位置**：元素 `x` 的目標索引為 `x - 1`（若 1-based）或 `x`（若 0-based）。
3. **交換策略**：每次把當前元素放到正確位置，再處理被換出來的元素，直到形成一個「環」。

### 執行步驟

1. 從索引 0 開始。
2. 若當前元素已在正確位置（`arr[i] == i + 1`），則移到下一個索引。
3. 若不在正確位置，將它與目標位置的元素交換。
4. 交換後，留在當前位置的元素可能仍不對，繼續處理，直到形成一個「環」結束。
5. 重複以上步驟直到整個陣列都走過。

### 範例

陣列 `[3, 1, 2]`（1-based，長度 3）：

- 索引 0：`3` 應在索引 2，與 `arr[2]=2` 交換 → `[2, 1, 3]`
- 索引 0：`2` 應在索引 1，與 `arr[1]=1` 交換 → `[1, 2, 3]`
- 索引 0：`1` 已在正確位置 → 完成
- 結果：`[1, 2, 3]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 陣列元素為 1～n 或 0～n-1 | 連續且不重複 |
| 找缺失數字 | 排序後可依索引判斷缺失 |
| 找重複數字 | 依索引與值判斷重複 |
| 找第一個缺失的正整數 | 常用於 LeetCode 題型 |
| 原地排序 | 不允許額外空間時 |

### 常見題型

1. **LeetCode 268**: Missing Number
2. **LeetCode 287**: Find the Duplicate Number
3. **LeetCode 41**: First Missing Positive
4. **LeetCode 442**: Find All Duplicates in an Array
5. **LeetCode 645**: Set Mismatch

---

## 四、時間複雜度

- **時間複雜度**：O(n)
- 每個元素最多被交換一次到正確位置，之後不再被動到。
- 雖然有巢狀迴圈，但每個元素最多被處理一次，整體為 O(n)。

---

## 五、空間複雜度

- **空間複雜度**：O(1)
- 只使用交換，不需要額外陣列或遞迴呼叫堆疊。
- 屬於原地排序。

---

## 六、與其他排序的比較

| 演算法 | 時間複雜度 | 空間複雜度 | 適用條件 |
|--------|------------|------------|----------|
| Cyclic Sort | O(n) | O(1) | 元素為 1～n 且不重複 |
| Quick Sort | O(n log n) 平均 | O(log n) | 一般陣列 |
| Merge Sort | O(n log n) | O(n) | 一般陣列 |
| Counting Sort | O(n + k) | O(k) | 元素範圍有限 |

---

## 七、總結

Cyclic Sort 適合「元素在 1～n 且不重複」的陣列，在 O(n) 時間、O(1) 空間內完成排序，常用於找缺失、重複或第一個缺失正整數等題型。若元素範圍不固定，則應改用 Quick Sort、Merge Sort 等其他排序方式。
