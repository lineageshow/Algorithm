# Bubble Sort 演算法詳解

## 一、什麼是 Bubble Sort？

**Bubble Sort（氣泡排序）** 是一種簡單的比較排序演算法。  
核心想法是：重複遍歷陣列，比較相鄰元素，若順序錯誤則交換，較大的元素會像「氣泡」一樣逐漸「浮」到陣列末端。每輪遍歷可確定一個元素的最終位置。

---

## 二、演算法原理

### 基本概念

1. **相鄰比較**：從陣列開頭開始，依序比較 `arr[i]` 與 `arr[i+1]`。
2. **交換策略**：若 `arr[i] > arr[i+1]`（升序），則交換兩者。
3. **每輪結果**：每輪遍歷後，未排序區中最大的元素會「浮」到正確位置。
4. **優化**：若某輪沒有發生任何交換，表示已排序完成，可提前結束。

### 執行步驟

1. 從索引 0 開始，比較 `arr[i]` 與 `arr[i+1]`。
2. 若 `arr[i] > arr[i+1]`，交換兩元素。
3. 索引加 1，重複步驟 1～2，直到索引到達 `n - 2`（倒數第二個）。
4. 完成一輪後，最後一個位置已確定，下一輪只需遍歷到 `n - 3`。
5. 重複上述步驟，直到某輪無交換或全部排序完成。

### 範例

陣列 `[5, 2, 8, 1, 9]`（升序）：

- 第 1 輪：`[2, 5, 1, 8, 9]`（5↔2, 8↔1, 9 已到位）
- 第 2 輪：`[2, 1, 5, 8, 9]`（5↔1）
- 第 3 輪：`[1, 2, 5, 8, 9]`（2↔1）
- 第 4 輪：無交換 → 完成
- 結果：`[1, 2, 5, 8, 9]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 教學與入門 | 邏輯簡單，易於理解排序概念 |
| 小規模資料 | n 很小時，實作簡單且效能尚可 |
| 幾乎已排序 | 搭配「無交換則提前結束」優化時較快 |
| 實作簡單 | 程式碼短，除錯容易 |
| 不建議大規模 | 大陣列效能差，實務上少用 |

### 常見題型

1. **LeetCode 75**: Sort Colors（可簡化為兩次遍歷）
2. **教學範例**：介紹排序演算法的基礎
3. **面試題**：要求實作最簡單的排序

---

## 四、時間複雜度

- **最佳時間複雜度**：O(n)，發生在陣列已排序且使用「無交換提前結束」優化時。
- **平均/最壞時間複雜度**：O(n²)
- 共需約 n 輪，每輪比較 O(n) 次，總比較次數約 n(n-1)/2。

---

## 五、空間複雜度

- **空間複雜度**：O(1)
- 只使用變數進行交換，不需額外陣列。
- 屬於原地排序。

---

## 六、與其他排序的比較

| 演算法 | 時間複雜度 | 空間複雜度 | 適用條件 |
|--------|------------|------------|----------|
| Bubble Sort | O(n²) / O(n) 最佳 | O(1) | 小規模、教學用 |
| Quick Sort | O(n log n) 平均 | O(log n) | 一般陣列 |
| Merge Sort | O(n log n) | O(n) | 一般陣列 |
| Heap Sort | O(n log n) | O(1) | 一般陣列、原地 |
| Cyclic Sort | O(n) | O(1) | 元素為 1～n 且不重複 |

---

## 七、總結

Bubble Sort 是最基礎的排序演算法之一，實作簡單但效能較差，時間複雜度為 O(n²)。適合用於教學或小規模資料。實務上排序大陣列時，應改用 Quick Sort、Merge Sort 或 Heap Sort。
