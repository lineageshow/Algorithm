# Quick Sort 演算法詳解

## 一、什麼是 Quick Sort？

**Quick Sort（快速排序）** 是一種基於分治法（Divide and Conquer）的比較排序演算法。  
核心想法是：選定一個基準元素（pivot），將陣列分成「小於基準」與「大於基準」兩部分，再對兩部分遞迴排序，最後合併即完成排序。

---

## 二、演算法原理

### 基本概念

1. **Pivot 選擇**：通常取第一個、最後一個、中間元素或隨機元素作為基準。
2. **Partition（分割）**：將陣列重新排列，使 pivot 左側元素皆小於等於 pivot，右側皆大於 pivot。
3. **遞迴**：對 pivot 左右兩側子陣列分別進行 Quick Sort，直到子陣列長度為 1 或 0。

### 執行步驟

1. 若陣列長度 ≤ 1，直接回傳（已排序）。
2. 選擇一個 pivot（例如最後一個元素）。
3. 執行 Partition：將小於 pivot 的元素移到左側，大於的移到右側，pivot 放到正確位置。
4. 對 pivot 左側子陣列遞迴執行 Quick Sort。
5. 對 pivot 右側子陣列遞迴執行 Quick Sort。
6. 合併後即為排序結果（原地操作，無需額外合併步驟）。

### 範例

陣列 `[5, 2, 9, 1, 7, 6]`，以最後一個元素 `6` 為 pivot：

- Partition：`[5, 2, 1]` | `6` | `[9, 7]`
- 左側 `[5, 2, 1]` 以 `1` 為 pivot → `[1]` | `2` | `[5]` → `[1, 2, 5]`
- 右側 `[9, 7]` 以 `7` 為 pivot → `[7]` | `9` → `[7, 9]`
- 結果：`[1, 2, 5, 6, 7, 9]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 一般陣列排序 | 最常用的通用排序演算法之一 |
| 大規模資料 | 平均效能佳，實務上常優於 Merge Sort |
| 原地排序需求 | 可實作為 O(log n) 空間的原地排序 |
| 不穩定排序可接受 | Quick Sort 為不穩定排序 |
| 隨機化 pivot | 可降低最壞情況發生機率 |

### 常見題型

1. **LeetCode 912**: Sort an Array
2. **LeetCode 215**: Kth Largest Element in an Array（可結合 Quick Select）
3. **LeetCode 75**: Sort Colors（荷蘭國旗問題，三路分割）
4. **LeetCode 148**: Sort List（鏈結串列排序）

---

## 四、時間複雜度

- **最佳/平均時間複雜度**：O(n log n)
- **最壞時間複雜度**：O(n²)，發生在 pivot 每次都選到最小或最大元素（例如已排序陣列且固定選邊界）
- 透過隨機選擇 pivot 或三數取中法，可大幅降低最壞情況機率。

---

## 五、空間複雜度

- **空間複雜度**：O(log n)（遞迴呼叫堆疊）
- 若實作為原地 Partition，不需額外陣列。
- 最壞情況下遞迴深度為 O(n)，空間變為 O(n)。

---

## 六、與其他排序的比較

| 演算法 | 時間複雜度 | 空間複雜度 | 適用條件 |
|--------|------------|------------|----------|
| Quick Sort | O(n log n) 平均 / O(n²) 最壞 | O(log n) | 一般陣列 |
| Cyclic Sort | O(n) | O(1) | 元素為 1～n 且不重複 |
| Merge Sort | O(n log n) | O(n) | 一般陣列、需穩定排序 |
| Counting Sort | O(n + k) | O(k) | 元素範圍有限 |

---

## 七、總結

Quick Sort 是實務上最廣泛使用的排序演算法之一，平均 O(n log n) 且可原地實作。需注意最壞 O(n²) 的情況，可透過隨機化或改良 pivot 選擇策略來改善。適合一般陣列排序，不適合元素範圍已知且較小的情況（此時 Counting Sort 可能更佳）。
