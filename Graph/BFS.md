# BFS 演算法詳解

## 一、什麼是 BFS？

**BFS（Breadth-First Search，廣度優先搜尋）** 是一種圖與樹的遍歷演算法，從起點開始**逐層**向外擴展，先處理完同一層的所有節點，再處理下一層。  
核心想法是：使用**佇列（Queue）**依序處理節點，保證「先加入的節點先被處理」，從而實現按距離（或層級）由近到遠的遍歷，適合求**最短路徑**、**層級遍歷**等問題。

---

## 二、演算法原理

### 基本概念

1. **佇列**：FIFO（先進先出），確保同層節點依序處理。
2. **已訪問標記**：避免重複訪問，通常用 `visited` 集合或陣列。
3. **層級概念**：每層對應「從起點出發的距離」，BFS 自然按層級順序遍歷。
4. **無權圖最短路徑**：在無權圖中，BFS 第一次到達某節點即為最短路徑。

### 常見變體

| 變體 | 說明 | 典型應用 |
|------|------|----------|
| **單源 BFS** | 從單一起點開始 | 最短路徑、層級遍歷 |
| **多源 BFS** | 多個起點同時入隊 | 多點同時擴散、0-1 矩陣 |
| **雙向 BFS** | 從起點與終點同時搜尋 | 縮小搜尋空間 |

### 執行步驟

1. 將起點加入佇列，並標記為已訪問。
2. 若佇列非空，取出隊首節點 `cur`。
3. 對 `cur` 的每個未訪問鄰居 `neighbor`：
   - 標記為已訪問。
   - 加入佇列。
   - 若需記錄距離，則 `dist[neighbor] = dist[cur] + 1`。
4. 重複步驟 2～3 直到佇列為空。
5. 若需層級遍歷，可每層開始前記錄當前佇列長度，一次處理整層。

### 範例：二元樹層級遍歷

```
      3
     / \
    9  20
       / \
      15  7
```

- 層 0：`[3]`
- 層 1：`[9, 20]`
- 層 2：`[15, 7]`
- 結果：`[[3], [9, 20], [15, 7]]`

## 三、使用情境

適合以下情境：

| 情境 | 說明 |
|------|------|
| 最短路徑（無權圖） | 第一次到達即最短路徑 |
| 層級遍歷 | 二元樹 Level Order、圖的層級 |
| 最少步數 | 如迷宮最短路徑、字串轉換最少步數 |
| 多點同時擴散 | 多源 BFS，如 0-1 矩陣 |
| 拓撲排序 | 可搭配 BFS（Kahn 演算法） |

## 四、時間複雜度

- **時間複雜度**：O(V + E)
  - V：節點數（頂點數）
  - E：邊數
- 每個節點最多入隊、出隊各一次，每條邊最多被檢查一次。
- 二元樹：O(n)，n 為節點數。
- 矩陣：O(m × n)，m、n 為行列數。

---

## 五、空間複雜度

- **空間複雜度**：O(V) 或 O(W)
  - 佇列最壞可存放 O(V) 個節點（如二元樹最底層）。
  - 二元樹：O(W)，W 為最大層寬，最壞 O(n)。
  - `visited` 陣列/集合：O(V)。

---

## 六、與其他技巧的比較

| 技巧 | 時間複雜度 | 空間複雜度 | 適用條件 |
|------|------------|------------|----------|
| BFS | O(V + E) | O(V) | 最短路徑、層級遍歷 |
| DFS | O(V + E) | O(V) 堆疊 | 路徑存在、拓撲排序、連通分量 |
| Dijkstra | O((V+E) log V) | O(V) | 有權圖最短路徑 |
| 雙向 BFS | O(V + E) | O(V) | 起終點明確，縮小搜尋 |

---

## 七、總結

BFS 利用佇列實現逐層遍歷，在無權圖中可求最短路徑，在樹中可做層級遍歷。適合「最少步數」「層級順序」等問題。需注意佇列與 visited 的空間消耗，圖較大時可考慮雙向 BFS 縮小搜尋範圍。
